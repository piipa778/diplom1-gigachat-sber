# Вкусный дворик — новый фронт + CMS

Стек сохранён: **Node.js + Express + PostgreSQL**. Фронт полностью переделан под нежный кондитерский дизайн как на макетах: кремовый фон, розовые акценты, тёмно-шоколадные блоки, круглые карточки, волнистые разделители, адаптивная сетка.

## Роли

### Пользователь
- может зарегистрироваться и войти через форму авторизации;
- может оставить отзыв;
- данные пользователя и отзыв сохраняются в PostgreSQL.

### Администратор
- входит через `ADMIN_LOGIN` и `ADMIN_PASSWORD` из `.env`;
- может редактировать текст и картинки прямо на страницах сайта;
- может загружать новые изображения;
- видит пользователей, отзывы и историю редактирований в `/admin.html`;
- все изменения сохраняются в PostgreSQL.

## Страницы

- `index.html` — главная;
- `catalog.html` — каталог сладостей;
- `o-kompanii.html` — о кондитерской;
- `otzivy.html` — отзывы;
- `kontakty.html` — контакты;
- `catering.html` — кейтеринг;
- `admin.html` — админ-панель.

## Как запустить

1. Установить зависимости:

```bash
npm install
```

2. Создать `.env` по примеру `.env.example`:

```env
PORT=3000
DATABASE_URL=postgres://postgres:postgres@localhost:5432/vkusny_dvorik
ADMIN_LOGIN=admin
ADMIN_PASSWORD=admin123
JWT_SECRET=diplom_reviews_site_secret_2026
```

3. Создать БД PostgreSQL `vkusny_dvorik`.

4. Запустить сервер:

```bash
npm start
```

5. Открыть сайт:

```text
http://localhost:3000
```

Админка:

```text
http://localhost:3000/admin.html
```

## Как редактировать сайт администратору

1. Войти как администратор через кнопку «Войти».
2. На странице появится панель CMS.
3. Нажать «Редактировать».
4. Изменить текст прямо на странице.
5. Кликнуть по картинке, чтобы заменить её URL или загрузить файл.
6. Нажать «Сохранить».

Изменения записываются в таблицу `page_content`, а факт редактирования — в `admin_edit_logs`.

## Новое в версии с каталогом блюд

Добавлен раздел **«Блюда / каталог»** в админ-панели. Теперь администратор может не только редактировать существующий текст и картинки страниц, но и добавлять новые блюда в каталог, менять их описание, цену, категорию, вес и изображение. Все данные сохраняются в PostgreSQL в таблице `products`, а действия администратора фиксируются в `admin_edit_logs`.
