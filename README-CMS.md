# Логика сайта: гость и администратор

В проект добавлена простая CMS-логика без смены стека: Node.js + PostgreSQL.

## Роли

### Гость
- открывает страницы сайта;
- видит сохранённые администратором тексты и изображения;
- может оставить отзыв через существующий API `/api/reviews`.

### Администратор
- входит по логину и паролю;
- редактирует тексты прямо на страницах сайта;
- меняет изображения: можно указать путь/URL или загрузить картинку файлом;
- сохраняет изменения в PostgreSQL;
- после сохранения гости видят обновлённый контент.

## Вход администратора

Админ-панель: `/admin.html` или `/admin`

Данные задаются в `.env`:

```env
ADMIN_LOGIN=admin
ADMIN_PASSWORD=change_me
JWT_SECRET=change_this_long_random_secret
```

Если переменные не заданы, временно используется логин `admin` и пароль `admin123`, но для дипломного проекта лучше обязательно изменить пароль.

## Как редактировать контент

1. Запустить проект и открыть `/admin.html`.
2. Войти как администратор.
3. Выбрать страницу из списка.
4. Нажать кнопку «Редактировать» в нижней панели.
5. Изменить текст прямо на странице.
6. Кликнуть по картинке, чтобы заменить её.
7. Нажать «Сохранить».

## Где хранятся изменения

В PostgreSQL добавлена таблица:

```sql
CREATE TABLE IF NOT EXISTS page_content (
  slug VARCHAR(160) PRIMARY KEY,
  content JSONB NOT NULL DEFAULT '{}'::jsonb,
  updated_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);
```

Для каждой страницы хранится JSON с двумя группами:

- `texts` — изменённые текстовые блоки;
- `images` — изменённые изображения и alt-тексты.

## Что изменено в интерфейсе

- Убраны колонки «Основное:» и «Меню:» из футера.
- Добавлена нижняя панель администратора, которая видна только при входе или при открытии страницы с `?admin=1`.
- Гостям панель редактирования не показывается.
